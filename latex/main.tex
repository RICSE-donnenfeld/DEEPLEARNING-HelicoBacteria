% ---------- Abstract ----------
\begin{abstract}
\noindent
We use the Quiron Salut dataset of 245 gastric biopsies (immunohistochemically stained whole-slide images) to implement and compare two patch-level models for \textit{Helicobacter pylori} detection.
A patch-based pipeline processes $256\times256$ patches extracted along tissue borders.
Two approaches are implemented: (i) a supervised convolutional neural network (CNN) trained on expert-annotated patches, and (ii) an unsupervised autoencoder (AE) trained on healthy patches and used for anomaly detection via reconstruction error.
Both models are evaluated at patch level with 5-fold patient-level cross-validation and at patient level by aggregating patch predictions into a contamination ratio and applying a learned threshold; we report results on the cross-validation splits and on the independent Hold-out set (116 patients).
The CNN achieves stronger and more stable performance (patch-level F1 0.92, patient-level CV F1 0.95, Hold-out F1 0.81); the AE is competitive at patch level but shows higher variability and lower specificity at patient level, with one fold exhibiting a pronounced reconstruction-error scale shift.
\end{abstract}

\section*{Glossary}
\addcontentsline{toc}{section}{Glossary}
\label{sec:glossary}

\begin{description}
  \item[AE] Autoencoder. Unsupervised model trained on healthy patches; anomaly score is reconstruction error.
  \item[CNN] Convolutional neural network. Supervised patch-level classifier (healthy vs.\ bacteria).
  \item[Contamination ratio] For a patient, the fraction of patches predicted positive; $r_p = \frac{1}{N_p}\sum_{i=1}^{N_p} \hat{z}_i$.
  \item[CV] Cross-validation. Here: 5-fold patient-level CV (splits by patient, not by patch).
  \item[F1] F1 score. Harmonic mean of precision and recall: $2 \cdot \mathrm{precision} \cdot \mathrm{recall} / (\mathrm{precision} + \mathrm{recall})$.
  \item[FPR] False positive rate. Proportion of true negatives incorrectly predicted positive; $\mathrm{FP}/(\mathrm{TN}+\mathrm{FP})$.
  \item[Hold-out] Independent test set (116 patients), not used for training or threshold selection; evaluated only with trained models.
  \item[IHC] Immunohistochemistry. Staining used on the gastric biopsy slides.
  \item[Patch] A $256\times256$ pixel crop from a whole-slide image (here, along the tissue border).
  \item[Patch-level] Evaluation or model applied to individual patches (as opposed to a full patient).
  \item[Patient-level] Evaluation or prediction after aggregating patch outputs per patient (e.g.\ via contamination ratio and $\tau_{patient}$).
  \item[Precision] Among predicted positives, the fraction that are truly positive; $\mathrm{TP}/(\mathrm{TP}+\mathrm{FP})$.
  \item[Recall] Among true positives, the fraction correctly predicted positive; $\mathrm{TP}/(\mathrm{TP}+\mathrm{FN})$. Syn.\ sensitivity.
  \item[ROC] Receiver operating characteristic. Curve of true positive rate vs.\ false positive rate as the decision threshold varies.
  \item[Specificity] Among true negatives, the fraction correctly predicted negative; $\mathrm{TN}/(\mathrm{TN}+\mathrm{FP})$.
  \item[TPR] True positive rate. Same as recall; $\mathrm{TP}/(\mathrm{TP}+\mathrm{FN})$.
  \item[$\tau_{patient}$] Patient-level threshold. A patient is predicted positive if their contamination ratio $r_p > \tau_{patient}$; chosen per fold on training patients (best F1).
  \item[$\tau_{patch}$] Patch-level threshold. A patch is predicted positive if its score (CNN probability or AE reconstruction error) exceeds $\tau_{patch}$; for the AE, chosen per fold on the validation set (ROC).
  \item[WSI] Whole-slide image. High-resolution digitised histology slide.
\end{description}

\section{Introduction}
\label{sec:intro}

\textit{Helicobacter pylori} is a gastric bacterium associated with chronic gastritis and an increased risk of gastric cancer. In clinical practice, detection often relies on the visual inspection of immunohistochemically (IHC) stained histopathology slides. Whole-slide images (WSI) can exceed 100\,000 pixels per side, making manual review time-consuming. This project investigates automated, patch-based detection: we classify small image patches as healthy or containing bacteria, then aggregate patch predictions to obtain a patient-level diagnosis.

The pipeline has three stages: (1) train a patch-level classifier on expert-annotated data; (2) apply it to all patches of each patient and compute the fraction of positive patches (contamination ratio); (3) compare that ratio to a threshold $\tau_{patient}$ to yield a binary patient-level diagnosis. Key challenges are the limited number of annotated patches, strong class imbalance (few patches contain bacteria), and the need to avoid data leakage by splitting at the patient level rather than the patch level.

We implement two patch-level models---a supervised CNN and an unsupervised autoencoder used for anomaly detection---and evaluate the full pipeline: 5-fold patient-level cross-validation for patch-level training, then patient-level aggregation with fold-consistent models and thresholds, evaluated on both the CV validation folds and the independent Hold-out set (116 patients). This report presents the methodology, implementation, experimental design, and results for the complete pipeline.

\section{Dataset Description}
\label{sec:dataset}

We use the Quiron Salut dataset: IHC-stained gastric biopsy whole-slide images from which $256\times256$ pixel patches have been cropped along the tissue border. Patient-level labels indicate H.\textit{ pylori} density: NEGATIVE, LOW, or HIGH. For binary classification we map NEGATIVE to 0 and (LOW, HIGH) to 1.

Patch-level annotations are provided in \texttt{CoordAnnotatedAllPatches.csv} with a \texttt{Presence} field: $-1$ (healthy), $1$ (bacteria present), or $0$ (unclear, which we excluded from training as there are 3 or 4 cases only).

The data are organised into three parts: \textbf{CrossValidation/Annotated} (expert-labelled patches for training and validating patch-level models), \textbf{CrossValidation/Cropped} (all patches per patient for aggregation experiments), and \textbf{Hold-out} (116 independent patients reserved for final evaluation only). \section{Methodology}
\label{sec:methodology}

\subsection{Pipeline overview}
The pipeline has three steps.
First, a patch-level model (CNN or autoencoder) is trained on annotated or healthy-only patches, with 5-fold patient-level cross-validation so that each fold yields a trained model and (for the AE) a patch-level threshold. Second, each fold model is applied to every patch of each patient (CV and Hold-out); the fraction of patches predicted positive per patient defines the contamination ratio $r_p$. Third, a patient-level threshold $\tau_{patient}$ is chosen per fold on the fold's training patients (maximising F1), then applied to validation and Hold-out patients to obtain binary patient-level predictions. We compare the CNN and the autoencoder at both patch level and patient level, reporting metrics on the CV folds and on the independent Hold-out set. Figure~\ref{fig:methodology} summarises the workflow.

\subsection{Preprocessing}
Patches are loaded as RGB images, resized to $256\times256$ if needed, and normalised per channel with $x' = (x - \mu)/\sigma$ where $\mu = \sigma = (0.5, 0.5, 0.5)$, so that inputs lie approximately in $[-1, 1]$. For the CNN we apply data augmentation: random flips, small rotations, and optional colour jitter.

\subsection{Patch-level approach 1: CNN classifier}
We use a lightweight CNN with three convolutional blocks followed by fully connected layers. Training minimises cross-entropy loss. With one-hot label $y$ and predicted class probabilities $\hat{p}_c$ (softmax over logits), the loss is
\[
\mathcal{L}_{\mathrm{CE}} = -\sum_{c \in \{0,1\}} y_c \log \hat{p}_c,
\]
with $c=0$ for healthy and $c=1$ for patches containing H.\textit{ pylori}. The model is trained with Adam; early stopping is optional. At inference, a patch is predicted positive if the probability of class 1 exceeds 0.5.

\subsection{Patch-level approach 2: Autoencoder anomaly detection}
The autoencoder is trained only on healthy patches and learns to reconstruct normal tissue. Training minimises mean squared error between input and reconstruction:
\[
\mathcal{L}_{\mathrm{AE}} = \frac{1}{CHW} \sum_{c,i,j} (x_{c,i,j} - \hat{x}_{c,i,j})^2,
\]
where $C=3$, $H=W=256$. The per-patch anomaly score $s(x)$ is the reconstruction error. We use either the mean over all pixels or the maximum over $8\times8$ spatial cell means (\texttt{max\_local}) so that small anomalous regions contribute more. A patch is predicted positive if $s(x) > \tau_{patch}$. The threshold $\tau_{patch}$ is chosen on the validation set as the point on the ROC curve closest to (FPR$=0$, TPR$=1$), implemented as \texttt{roc\_threshold\_optimal}.

\subsection{Patient-level aggregation}
For each patient $p$ with $N_p$ patches and binary patch predictions $\hat{z}_i \in \{0,1\}$, we define the contamination ratio
\[
r_p = \frac{1}{N_p}\sum_{i=1}^{N_p} \hat{z}_i.
\]
The patient is predicted positive if $r_p > \tau_{patient}$. In our implementation, $\hat{z}_i$ is derived from a patch-level score and threshold:
\begin{itemize}
  \item CNN: score = $P(\mathrm{unhealthy})$, threshold $\tau_{patch}^{cnn}=0.5$ (default).
  \item AE: score = reconstruction error (\texttt{mean} or \texttt{max\_local}), threshold $\tau_{patch}^{ae}$ from ROC (\texttt{autoencoder\_threshold.txt} or fold-specific threshold files).
\end{itemize}
The patient threshold $\tau_{patient}$ is selected on training patients by maximising patient-level F1 over candidate ratio thresholds, then evaluated on validation patients. The same protocol is applied fold-by-fold in patient-level CV, and then on Hold-out.

\section{Implementation Details}
\label{sec:implementation}

\subsection{Development iterations}
The CNN started as a minimal classifier: three small convolutional blocks (3$\to$16$\to$32$\to$64 channels, ReLU, max pooling) and two fully connected layers mapping the flattened 64$\times$32$\times$32 features to two class logits. No architectural changes were required for good performance; the main refinements were patient-level train/validation splitting to avoid leakage, ignoring patches with \texttt{Presence=0} in the annotation file, and training for more epochs (e.g.\ 150 in the Slurm script). The current design remains lightweight and fast to train.

The autoencoder began with a shallower encoder (32$\to$64$\to$128$\to$256$\to$256 channels, 8$\times$8 bottleneck) and ReLU activations. To avoid losing information in the bottleneck (ReLU zeros negatives), ReLU was replaced by LeakyReLU. The encoder and decoder were then deepened and widened (64$\to$128$\to$256$\to$512$\to$512 channels, 8$\times$8$\times$512 bottleneck) to improve reconstruction of healthy tissue so that anomalous patches yield higher error. The anomaly score was extended with a \texttt{max\_local} option (maximum over $8\times8$ spatial cell means) so that focal H.\textit{ pylori} regions drive the score. Checkpoint selection was fixed to keep the best model by validation loss instead of the last epoch, and the default aggregation and epoch count were set to \texttt{max\_local} and 35 (overridable via Slurm).

\subsection{Code structure}
Core scripts:
\begin{itemize}
  \item \texttt{model\_classifier.py}: CNN definition, training, and evaluation.
  \item \texttt{model\_autoencoder.py}: AE training and ROC-based patch threshold selection.
  \item \texttt{src/helico/cv\_utils.py}: shared patient-level split and metric helpers.
  \item \texttt{compare\_patch\_classifiers.py}: side-by-side patch-level comparison.
\end{itemize}

Patient-level diagnosis is implemented in \texttt{patient\_level\_pipeline.py}. The script supports:
\begin{itemize}
  \item single-checkpoint mode;
  \item fold-consistent mode (\texttt{--use-cv-fold-checkpoints}) with models from \texttt{cv/cnn/fold\_i/} and \texttt{cv/autoencoder/fold\_i/}.
\end{itemize}
Outputs are written under \texttt{output/patient\_level/}: fold metrics, Hold-out-per-fold metrics, patient-level detail rows, and summary statistics.

Cluster scripts:
\begin{itemize}
  \item training: \texttt{train\_classifier.sbatch.sh}, \texttt{train\_autoencoder.sbatch.sh} (legacy \texttt{train.sbatch.sh});
  \item patient-level: \texttt{patient\_cnn\_fold.sbatch.sh}, \texttt{patient\_autoencoder\_fold.sbatch.sh}, \texttt{patient\_plots.sbatch.sh};

    \item \texttt{analyze\_ae\_fold\_diagnostics.py}

  \end{itemize}
\end{itemize}

\section{Experimental Design}
\label{sec:experiments}

We split data at the \textbf{patient level} so that a patient never appears in both train and validation folds. The current evaluation uses \textbf{5-fold patient-level cross-validation} (\texttt{--k-folds 5}, \texttt{seed=42}) for both CNN and AE. For each fold, checkpoints and metrics are stored under \texttt{cv/cnn/fold\_i/} and \texttt{cv/autoencoder/fold\_i/}. The Hold-out set (116 patients) remains reserved for a single final evaluation after all model and threshold design choices are fixed.

For patient-level diagnosis, we compute contamination ratios on \texttt{CrossValidation/Cropped} and select fold-wise $\tau_{patient}$ from training patients only (best F1). We then evaluate on validation patients and additionally report Hold-out metrics per fold using the same fold model and $\tau_{patient}$. This avoids mixing fold artefacts and keeps patch-level and patient-level protocols consistent.

We report fold-wise and aggregate patch-level metrics (accuracy, precision, recall, specificity, F1), using both mean$\pm$std and robust summaries (median/IQR). For AE we additionally report threshold behaviour, TPR and FPR by fold.

% ==========================================================
\section{Discussion}
\label{sec:discussion}

With patient-level 5-fold CV, the CNN consistently outperforms the AE in accuracy/precision/F1 and exhibits lower fold-to-fold variance. The AE remains useful in low-label settings but is notably more sensitive to fold-specific data distribution.

The main observed limitation is threshold instability under distribution shift: fold 3 presents an upward shift of reconstruction errors for both classes, which inflates local threshold and degrades specificity at patient level. A pooled global threshold improved some folds but harmed specificity severely on fold 3, so a single global threshold is not yet robust enough. The current fold-consistent patient pipeline makes this behaviour explicit and reproducible, and provides a solid baseline for future threshold normalisation/calibration strategies (e.g., robust scaling or nested calibration).

% ==========================================================
\section{Conclusion}
\label{sec:conclusion}
We implemented and evaluated a full pipeline for H.\textit{ pylori} detection in IHC-stained gastric biopsies: two patch-level models (supervised CNN and unsupervised autoencoder for anomaly detection), trained with 5-fold patient-level cross-validation, and a patient-level aggregation stage that computes contamination ratios and applies fold-specific thresholds, evaluated on both the CV validation folds and the independent Hold-out set.

At patch level, the CNN achieves stronger and more stable performance (F1 0.9201$\pm$0.0382) than the AE (F1 0.8397$\pm$0.0696). At patient level, the CNN again outperforms (CV F1 0.947, Hold-out F1 0.81), while the AE maintains high sensitivity but lower specificity (CV F1 0.82, Hold-out F1 0.72). AE fold diagnostics identify a fold-specific error-scale anomaly (fold 3), motivating robust threshold calibration rather than a single pooled threshold.

\paragraph{Possible continuation.}
Several directions could extend this work:
(i)~\textbf{Threshold calibration for the AE}---e.g.\ robust scaling or per-fold normalisation of reconstruction errors so that a single or more stable threshold can be used across folds and specificity improves without losing sensitivity;
(ii)~\textbf{More annotated data}---the current patch set is limited and imbalanced; additional expert labels would help both the CNN and the AE;
(iii)~\textbf{Architecture and training}---deeper or pre-trained patch encoders (e.g.\ ResNet, vision transformers), and optional class weighting or focal loss to handle imbalance;

class SimpleCNN(nn.Module):
    def __init__(self, num_classes: int = 2) -> None:
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 16, 3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
            nn.Conv2d(16, 32, 3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
            nn.Conv2d(32, 64, 3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
        )
        self.classifier = nn.Sequential(
            nn.Flatten(), nn.Linear(64 * 32 * 32, 128), nn.ReLU(),
            nn.Linear(128, num_classes),
        )
    def forward(self, x): return self.classifier(self.features(x))
\end{lstlisting}

\begin{lstlisting}[language=Python, caption={Autoencoder: Encoder, Decoder, and full model (\texttt{model\_autoencoder.py}).},label=lst:ae]
class Encoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv = nn.Sequential(
            nn.Conv2d(3, 64, 3, padding=1), nn.LeakyReLU(0.2), nn.MaxPool2d(2),
            nn.Conv2d(64, 128, 3, padding=1), nn.LeakyReLU(0.2), nn.MaxPool2d(2),
            nn.Conv2d(128, 256, 3, padding=1), nn.LeakyReLU(0.2), nn.MaxPool2d(2),
            nn.Conv2d(256, 512, 3, padding=1), nn.LeakyReLU(0.2), nn.MaxPool2d(2),
            nn.Conv2d(512, 512, 3, padding=1), nn.LeakyReLU(0.2), nn.MaxPool2d(2),
        )
    def forward(self, x): return self.conv(x)

class Decoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.up = nn.Sequential(
            nn.ConvTranspose2d(512, 512, 2, stride=2), nn.LeakyReLU(0.2),
            nn.ConvTranspose2d(512, 256, 2, stride=2), nn.LeakyReLU(0.2),
            nn.ConvTranspose2d(256, 128, 2, stride=2), nn.LeakyReLU(0.2),
            nn.ConvTranspose2d(128, 64, 2, stride=2), nn.LeakyReLU(0.2),
            nn.ConvTranspose2d(64, 3, 2, stride=2), nn.Tanh(),
        )
    def forward(self, x): return self.up(x)

class PatchAutoencoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.encoder = Encoder()
        self.decoder = Decoder()
    def forward(self, x):
        return self.decoder(self.encoder(x))
\end{lstlisting}

\begin{lstlisting}[language=Python, caption={AE per-sample reconstruction error (\texttt{model\_autoencoder.py}). \texttt{mean}: pixel MSE; \texttt{max\_local}: max over 8$\times$8 cell means.},label=lst:ae_error]
def reconstruction_error_per_sample(model, x, aggregation="mean"):
    x_hat = model(x)
    err = (x - x_hat).pow(2).mean(dim=1, keepdim=True)  # (B, 1, H, W)
    if aggregation == "mean":
        return err.view(x.size(0), -1).mean(dim=1)
    if aggregation == "max_local":
        B, _, H, W = err.shape
        cell = 8
        nc_h, nc_w = H // cell, W // cell
        err = err.view(B, 1, nc_h, cell, nc_w, cell).mean(dim=(3, 5))
        return err.view(B, -1).max(dim=1)[0]
\end{lstlisting}

\begin{lstlisting}[language=Python, caption={ROC-based patch threshold: closest to (FPR=0, TPR=1) (\texttt{roc\_threshold\_optimal}, \texttt{model\_autoencoder.py}).},label=lst:roc]
def roc_threshold_optimal(errors, labels, positive_class=1):
    order = np.argsort(-errors)
    sorted_errors, sorted_labels = errors[order], (labels[order] == positive_class).astype(np.float64)
    n_pos, n_neg = sorted_labels.sum(), len(sorted_labels) - n_pos
    tpr = np.cumsum(sorted_labels) / n_pos
    fpr = np.cumsum(1.0 - sorted_labels) / n_neg
    thresh = np.concatenate([[sorted_errors[0]+1], sorted_errors, [sorted_errors[-1]-1]])
    thresh = (thresh[:-1] + thresh[1:]) / 2
    dist = fpr**2 + (1 - tpr)**2
    idx = np.argmin(dist)
    return float(thresh[idx]), float(tpr[idx]), float(fpr[idx])
\end{lstlisting}
